# 背景

移动端登录页面，用户需要输入账号和密码完成登录。

# 问题描述

密码输入框没有提供「显示/隐藏密码」的切换按钮，用户无法确认自己输入的密码是否正确。

## 发现的问题

| 项目 | 内容 |
|------|------|
| **问题描述** | 密码输入框缺少显示/隐藏密码切换功能 |
| **风险等级** | 🟡 中 |
| **涉及元素** | 登录页 - 密码输入框 |
| **对用户体验的影响** | 用户无法确认输入内容，导致：1) 输入错误后需要多次尝试 2) 增加用户焦虑感 |

## 数据支撑

- 密码输入错误率：**23%**（行业平均 8-12%）
- 用户首次登录成功率：**71%**（目标 > 90%）
- 客服反馈中"忘记密码/登录失败"占比：**15%**

# 解决方案

在密码输入框右侧添加眼睛图标，支持切换密码可见性。

## 设计要点

1. **图标设计**：使用标准眼睛图标 👁️ / 👁️‍🗨️
2. **位置**：输入框内部右侧
3. **交互**：点击切换，无需长按
4. **默认状态**：隐藏（圆点显示）
5. **触摸目标**：>= 44x44pt

## 平台规范参考

- iOS Human Interface Guidelines 推荐提供密码可见性切换
- Material Design Text Fields 规范明确要求

# 效果数据

优化后：
- 密码输入错误率：**23% → 9%**（下降 61%）
- 首次登录成功率：**71% → 92%**（提升 30%）
- 相关客服工单：**下降 40%**

# 经验总结

1. **密码输入框必须提供显示/隐藏切换** - 这是基本的可用性要求
2. **符合平台规范** - iOS 和 Android 都有明确指导
3. **数据驱动** - 通过数据发现问题，用数据验证效果
4. **状态反馈维度** - 这属于"状态反馈与操作确认"维度的典型问题
